# ADR: opt-in wait_for_capture_ready

**Статус:** принято  
**Дата:** 2026-05-20

## Контекст

Сервис historically использовал `networkidle2` и не ждал сигнала готовности SPA. Дашборды загружают отчёты асинхронно; ранний скриншот давал пустые или неполные кадры.

## Решение

- Поле **`wait_for_capture_ready: boolean`** в теле `/convert`, `/convert-and-send`, `/jobs`.
- **`false` / отсутствует** — legacy-ветка без изменения контракта для старых клиентов.
- **`true`** — после `goto` ожидание `window.__CAPTURE_READY__ === true`, затем widget clip (если задан); структурированные ошибки (`dashboard_not_ready`, …).
- Оркестратор для дашбордов **обязан** передавать `true`.

## Последствия

- Обратная совместимость сохранена.
- Ключевой гейт — флаг frontend, не глобальная смена `waitUntil` для всех клиентов.
- Страницы без флага при `true` получат `dashboard_not_ready`.

## Отклонённые альтернативы

- **Включить ожидание ready по умолчанию** — сломает legacy-клиентов.
- **Заменить networkidle2 глобально** — риск зависаний на сторонних URL.
